# 后台管理面板

## 架构说明

```
┌─────────────────────────────────────────────────────────┐
│                   http://localhost:8000                  │
├─────────────────────────────────────────────────────────┤
│                    FastAPI 服务                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐ │
│  │   API 接口  │  │  WebSocket  │  │  前端静态文件    │ │
│  │  /api/*     │  │    /ws      │  │  (Vue SPA)      │ │
│  └─────────────┘  └─────────────┘  └─────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**统一入口**：一个端口同时提供 API 和前端页面

## 快速开始

### 方式1: 一键构建 (推荐)

```bash
# Windows PowerShell
.\build-frontend.ps1

# Windows 批处理
build-frontend.bat
```

### 方式2: 手动构建

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装依赖
npm install

# 3. 构建
npm run build

# 4. 返回项目根目录
cd ..
```

### 方式3: 开发模式 (前后端分离)

```bash
# 终端1: 启动后端
python main.py

# 终端2: 启动前端开发服务器
cd frontend
npm run dev
# 访问 http://localhost:3000
```

## 启动服务

```bash
# 方式1: 启动核心服务 (包含 API + 策略引擎)
python main.py

# 方式2: 仅启动 API 服务
uvicorn api:app --port 8000
```

## 访问地址

| 地址 | 说明 |
|------|------|
| http://localhost:8000 | 管理面板 (需先构建前端) |
| http://localhost:8000/docs | Swagger API 文档 |
| http://localhost:8000/redoc | ReDoc API 文档 |
| http://localhost:8000/api/* | API 接口 |
| http://localhost:8000/ws | WebSocket 连接 |

## 开发模式 vs 生产模式

| 特性 | 开发模式 | 生产模式 |
|------|----------|----------|
| 前端 | 独立开发服务器 (3000) | 构建后由后端托管 (8000) |
| 热更新 | ✅ 支持 | ❌ 需重新构建 |
| API 代理 | Vite 自动代理 | 直接访问 |
| 部署 | 两个服务 | 一个服务 |

## 常见问题

### Q: 访问 8000 端口显示"前端未构建"

A: 需要先构建前端：
```bash
cd frontend
npm install
npm run build
cd ..
```

### Q: 如何修改前端端口 (开发模式)

A: 编辑 `frontend/vite.config.ts`：
```typescript
server: {
  port: 3001,  // 修改为其他端口
}
```

### Q: 如何同时运行前后端

A: 打开两个终端：
```bash
# 终端1: 后端
python main.py

# 终端2: 前端
cd frontend && npm run dev
```
